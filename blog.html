  1. Difference between map(), filter(), forEach() and find() .

    The map() function transforms each element of the original array according to the function supplied, and returns a new array with the same length as the original array. The initial array is unaltered.

    Filter() creates a new array with just the elements that pass a specified condition (specified by a function). The initial array is unaltered.

    For each element in an array, the forEach() function runs the supplied function once. It does not modify the initial array or return anything.

    The value of the first element in an array that meets a testing function is returned by the find() method. If no element passes the test, it returns undef.

  2. Difference between localStorage , Session storage and Cookies .

    LocalStorage: Information saved in LocalStorage is permanent, meaning it is still accessible even after the browser or device is closed. A domain can only access LocalStorage, which has a maximum size of 5 to 10MB. With every HTTP request, data kept in LocalStorage is not transmitted to the server.

    SessionStorage: Temporary information is erased when the browser or tab is closed and is stored in SessionStorage. Similar in capacity to LocalStorage, SessionStorage is only accessible by the domain that created it. SessionStorage data is not transmitted to the server with every HTTP request, similar to LocalStorage.

    Cookies: The server stores a few lines of text known as cookies on the user's device. They can be used to store data like login credentials or user preferences and have a lifespan that is specified by the server. Cookies can be accessed by both the client and the server and have a maximum size of 4KB. Each HTTP request includes a cookie, which can cause the request/response cycle to lag.

  3. What is event loop ? How does it works ?

    Event loop, which is in charge of running the program, gathering and analyzing events, and carrying out queued sub-tasks.

    The Call Stack and the Callback Queue are two simple tasks performed by the Event Loop. The Event Loop will pull the first event from the queue and put it to the Call Stack if it is empty, where it is effectively run.
